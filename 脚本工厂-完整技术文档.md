# 宏达新媒体脚本工厂 — 脚本生成逻辑完整说明

## 一、整体架构

脚本工厂由以下核心模块组成：

```
lib/ai/
  ├── script-pipeline.ts    # 核心流水线（Hook生成、策略生成、脚本组装、流水线编排）
  ├── script-scoring.ts     # 评分引擎（8维扣分制评分卡）
  ├── prompts.ts            # AI提示词模板（供OpenAI API使用，当API不可用时用mock）
  ├── schemas.ts            # Zod验证Schema
  └── service.ts            # AI服务调用层（含mock降级）

components/scripts/
  ├── ScriptGeneratorWizard.tsx  # 4步生成向导UI
  ├── ScriptsContent.tsx         # 脚本工厂主逻辑
  ├── ScriptsRenderer.tsx        # 三栏工作台渲染
  ├── ScriptAIPane.tsx           # AI操作（润色、编辑、推进选题库）
  ├── ScriptListPane.tsx         # 左栏脚本列表
  ├── ScriptEditorPane.tsx       # 中栏脚本编辑器
  ├── ScoreCard.tsx              # 评分卡片展示
  └── BulkActionBar.tsx          # 批量操作

api/ai/
  ├── generate-script/route.ts          # 旧版AI生成（保留兼容）
  ├── generate-script-strategy/route.ts # 策略生成API
  ├── generate-script-draft/route.ts    # 草稿生成API
  ├── polish-script/route.ts            # 润色API
  ├── check-script-risk/route.ts        # 风险检查API
  ├── generate-script-pipeline/route.ts # 端到端流水线API
  └── score-script/route.ts             # 评分API
```

## 二、核心流水线（script-pipeline.ts v3）

### 2.1 关键接口

**ScriptStrategy（脚本策略卡）**
```
topic            — 脚本主题
hook             — 开头钩子（≤25字）
hookType         — 钩子类型：conflict/question/counterintuitive/scenario/data
targetCustomer   — 目标客户
customerPain     — 客户痛点
corePoint        — 核心观点（只讲一个）
whyWatch         — 用户为什么愿意看
solveWhat        — 这条视频解决什么问题
structure        — 内容结构
conversionGoal   — 转化目标（引导私信/评论/发图）
risksToAvoid     — 需要避免的风险
suitablePlatform — 适合平台
suggestedDuration— 建议时长
suggestedActing  — 建议出镜方式
```

**PipelineResult（流水线输出）**
```
strategy          — 策略卡
isBroad           — 输入是否宽泛
subTopics         — 拆解后的子选题（如果输入宽泛）
variants          — 3个时长变体（15s/30s/60s）
bestVariant       — 最优变体
risk              — 风险等级
recommendedStatus — 推荐状态（pending_review/draft/needs_rewrite/discard）
mock              — 是否mock模式
```

### 2.2 钩子生成规则（generateHook）

**禁止的开头：**
- 很多客户问我这个问题
- 今天统一回答一下
- 今天给大家讲一下
- 最近很多朋友问
- 大家都知道
- 在热转印行业中
- 随着市场发展

**钩子要求：**
1. 直接点出客户问题，不超过25字
2. 不用泛泛铺垫
3. 不出现"这个问题"
4. 最好包含数字、材料、场景、冲突或反常识

**9类钩子模板：**

| 类别 | 示例 |
|------|------|
| price | "500个杯子，真的不能做热转印吗？" |
| material | "PE瓶能不能做热转印？这句话不能直接回答。" |
| test | "你问我会不会掉，我最怕直接回答'不会'。" |
| color | "客户说颜色按图片做，这个风险很大。" |
| sample | "打样和大货做到一模一样，真的那么简单吗？" |
| small_batch | "500个小批量，适不适合热转印？" |
| misunderstanding | "客户只发图片，为什么判断不了能不能做热转印？" |
| process | "标准热转印花膜有几层？不是只看图案那么简单。" |
| anti_back | "防背粘只是多加一张纸？没那么简单。" |

### 2.3 宽泛主题拆解（splitBroadTopic）

如果用户输入的主题太宽泛（如"客户常见问题一次性说清楚"、"热转印介绍"），自动拆成5-10个具体子选题。

预定义拆解：

| 宽泛主题 | 拆解数量 |
|----------|----------|
| 客户常见问题 | 8个 |
| 热转印介绍 | 5个 |
| 注意事项 | 4个 |
| 其他（>15字） | 5个 |

拆解后的第一个子选题作为实际生成主题。

### 2.4 策略生成（generateScriptStrategy）

生成脚本前先生成策略卡：
1. 调用 splitBroadTopic 判断是否宽泛
2. 调用 generateHook 生成钩子
3. 从 account 信息提取目标客户、转化目标
4. 设置核心观点、用户为什么看、解决什么问题

### 2.5 时长变体生成（buildScriptByDuration）

**字数硬限制：**
```
15秒: 80-120字，只讲1个核心点
30秒: 150-220字，最多2个核心点
60秒: 280-420字，最多3个核心点
```

**脚本结构模板（chunk_templates）：**

每类内容有5个构建块：
```
hook   — 钩子（1-2个变体）
reason — 原因（2个变体，15秒挑1个）
scene  — 客户场景（2个变体，30秒以上加入）
action — 下一步动作（1-2个变体）
cta    — 转化引导（1个变体，60秒加入）
```

不同时长的组装方式：
```
15秒: hook + reason + action
30秒: hook + reason + scene + action  
60秒: hook + reason + scene + action + cta
```

**内容类型自动匹配（getChunkType）：**

根据 customerPain 和 knowledgeCards 自动匹配：
- 包含"价格/多少钱/报价" → price
- 包含"材质/PE/PP/ABS" → material
- 包含"测试/附着力/掉" → test
- 包含"颜色/色差" → color
- 包含"打样/样品" → sample
- 包含"误区/注意/常见" → misunderstanding
- knowledgeCard含"花膜结构" → process
- knowledgeCard含"防背粘" → anti_back
- 默认 → price

### 2.6 端到端流水线（runPipeline）

```
输入
  ↓
① 判断是否宽泛主题 → 是则拆解
  ↓
② 生成策略卡（strategy）
  ↓
③ 生成3个时长变体（15s/30s/60s），每个附带评分
  ↓
④ 按评分降序排序，选出最优变体
  ↓
⑤ 确定推荐状态（pending_review/draft/needs_rewrite/discard）
  ↓
输出 PipelineResult
```

### 2.7 AI味去除（removeAiTone）

删除24种禁止表达：
```
很多客户问我这个问题
今天统一回答一下
今天给大家讲一下
最近很多朋友问
大家都知道
在热转印行业中
随着市场发展
首先 / 其次 / 最后
综上所述 / 显而易见
有效提升 / 赋能 / 助力
专业解决方案 / 欢迎联系我们
一站式 / 全方位 / 闭环 / 矩阵
在这个视频里 / 今天在视频里
```

### 2.8 风险检查（checkScriptRisk）

7种风险模式检测：
```
一定能做/肯定能做/保证能做    → 乱承诺能做
保证不掉/绝对不会掉           → 乱承诺附着力
\d+元/\d+块钱               → 报具体价格
\d+天交货/一定交期            → 承诺交期
颜色一模一样/百分之百还原       → 乱承诺颜色
最能/最好/最专业/最便宜         → 绝对化表达
永不掉/永远不掉               → 乱承诺附着力
```

风险等级：低（0个风险）、中（1-2个）、高（3个以上）

### 2.9 批量生成（batchGenerateShortScripts）

从8个不同切口批量生成脚本：
1. 价格角度
2. 材质角度
3. 测试角度
4. 颜色角度
5. 打样角度
6. 小批量角度
7. 客户误区
8. 工艺角度

每条生成后自动评分，低于70分的不默认勾选。

### 2.10 批量保存规则（batchSaveScripts）

```
高风险 → 不能保存为待审核
<60分  → 不建议保存
≥60分  → 可保存为草稿
≥70分  → 可保存为草稿
≥85分+低风险 → 可保存为待审核
```

## 三、评分引擎（script-scoring.ts v2）

### 3.1 评分方式：扣分制

从100分开始，根据问题扣分。目标是"真人说话、讲一个客户问题、逻辑清楚"。

### 3.2 8个评分维度

| 维度 | 满分 | 扣分逻辑 |
|------|------|----------|
| 开头钩子 | 20 | 是否有具体客户问题/冲突/反常识？泛泛开头扣20 |
| 单点聚焦 | 15 | 用"第一/第二/第三"枚举3个点以上扣15 |
| 口语化程度 | 20 | 句子太长扣8；没有"我跟你说/说白了"扣6；像文章扣6 |
| 客户痛点清晰度 | 15 | 有没有"为什么/怎么/能不能"这类问题词？有没有提到客户或您 |
| 工厂判断逻辑 | 10 | 有没有"因为/所以/判断"这类推理词？有没有提到材质/工艺 |
| 时长控制 | 10 | 超字数扣15（先扣分） |
| 转化动作 | 5 | 结尾有没有"评论区/发产品/私信"等引导 |
| 风险表达安全 | 5 | 出现"保证/绝对/一定能做"扣15 |

### 3.3 额外扣分项

```
出现禁止开头短语          → -10/条
出现书面语/空话           → -5/个
像文章不是口播            → -20
没有具体客户问题开头      → -20
没有风险提醒             → -5
出现"第一/第二/第三"枚举  → -5/个
```

### 3.4 等级

```
≥90 → S级
≥80 → A级
≥70 → B级
≥60 → C级
≥40 → D级
<40 → F级
```

### 3.5 推荐状态

```
≥85 + 低风险 → 待审核（pending_review）
≥70          → 草稿（draft）
≥60          → 需重写（needs_rewrite）
<60          → 不建议保存（discard）
```

### 3.6 验证

文章式脚本"客户常见问题?今天给你说清楚。很多客户问我这个问题..."的评分应低于70分。

## 四、AI 提示词（prompts.ts）

### 4.1 主提示词（SYSTEM_PROMPT）

设定AI为"宏达新媒体作战中台"助手，服务于广东宏达印业新媒体部门。规则：
- 所有脚本必须匹配指定账号人设
- 禁止乱承诺价格、交期、附着力
- 必须基于知识库
- 输出JSON格式

### 4.2 脚本策略提示词（GENERATE_STRATEGY_PROMPT）

输入：账号信息 + 选题 + 知识卡
输出：策略卡JSON（topic/targetCustomer/customerPain/corePoint/hookDirection等）

### 4.3 脚本草稿提示词（GENERATE_DRAFT_PROMPT）

输入：策略卡 + 账号人设 + 模板
输出：完整口播稿
要求：
- 开头必须是客户真实场景，前3秒抓住人
- 中间讲清楚一个判断逻辑
- 语言像工厂老板/业务员
- 每句话不超过30字
- 结尾有客户下一步动作引导
- 禁止：首先/其次/最后/综上所述/显而易见/有效提升/赋能/助力

### 4.4 润色提示词（POLISH_SCRIPT_PROMPT）

输入：原脚本
输出：口语化版本
规则：
- 去文章感
- 每句话不超过20字
- 删除禁止词
- 用"我跟你说"、"说白了"、"你记住"连接

### 4.5 风险检查提示词（CHECK_RISK_PROMPT）

输入：脚本 + 知识卡
检查10类风险（乱承诺能做/价格/交期/附着力/颜色/保密/工艺/侵权/贬低同行/待审核知识）

### 4.6 评分提示词（SCORE_SCRIPT_PROMPT）

输入：脚本 + 账号人设
8维度评分（满分100），输出JSON带推荐状态

### 4.7 端到端流水线提示词（GENERATE_PIPELINE_PROMPT）

输入：账号 + 主题/痛点 + 时长 + 知识卡
5步执行：拆宽泛→生成钩子→生成策略→生成脚本→按时长变体
输出JSON包含isBroad/subTopics/strategy/variants/bestDuration

## 五、用户交互流程（ScriptGeneratorWizard）

### 5.1 4步向导

**第一步：账号平台**
- 选择账号（自动带入人设/定位/目标客户/转化目标）
- 选择平台（视频号/抖音）

**第二步：内容定位**
- 输入产品/工艺
- 输入客户痛点（可点击预设建议标签）
- 可选材质
- 检测宽泛主题 → 展示子选题供选择
- 实时预览钩子

**第三步：脚本参数**
- 视频时长（15s/30s/60s，每项标注字数范围）
- 脚本结构（系统自动选择）
- 出镜方式
- 语气风格

**第四步：参考知识**
- 按账号筛选知识卡
- 显示每张知识卡的标题/结论/对外范围

### 5.2 生成结果展示

点击"生成脚本"后展示：
1. **策略卡摘要** — 主题/目标客户/痛点/核心观点
2. **开头钩子** — 高亮展示
3. **3个时长变体** — 每个显示字数+评分+等级，可点击切换
4. **选中变体详情** — 完整脚本 + 评分优势 + 不足 + 风险提醒
5. **推荐状态** — 待审核/草稿/需重写/不建议保存

### 5.3 确认生成

点击"确认并添加到工作台"→ 脚本加入三栏工作台。

### 5.4 后续操作（ScriptAIPane）

- **推进选题库** — push到选题库待审核
- **编辑** — 微调脚本文字
- **删除** — 删除脚本
- **AI操作** — 12种润色选项（更口语/更专业/更短/视频号版/查风险等）
- **评分卡** — 实时展示评分
- **来源追溯** — 状态/推荐/风险/字数/时长

## 六、数据处理流

```
用户输入（在向导中选择）
  ↓
generateShortVideoScript(input)
  ├── generateHook(input)             → 生成 ≤25字钩子
  ├── generateScriptStrategy(input)   → 生成策略卡
  ├── getChunkType(input)             → 匹配内容类型
  ├── buildScriptByDuration(...)      → 组装脚本
  └── 返回 ShortVideoScript
  
  ↓
scoreScript(script, duration)         → ScriptScoreResult
  
  ↓
创建 Script 对象，加入scripts状态
  ↓
脚本显示在三栏工作台
  ↓
用户可：推进选题库 / 编辑 / 删除 / AI润色 / 查看评分
```

## 七、跨设备同步

脚本、选题等数据通过 Supabase 的 `site_data` 表实现跨设备同步。
- 客户端每3秒轮询 `/api/data?key=<key>` 获取最新数据
- 写入时同时保存到 localStorage 和 Supabase
- 时间戳比较冲突，只保留更新的版本
- 未配置 Supabase 时降级到文件存储

## 八、已知问题和待改进

1. **模板库有限** — 目前只有8套chunk模板，可扩展至15-20套
2. **随机性** — 钩子和模板目前随机选择，可改成按上下文智能匹配
3. **知识卡利用率** — 当前只用了第一张知识卡标题，未充分挖掘知识卡内容
4. **评分校准** — 需要真人标注一批脚本来校准评分权重
5. **AI模式（OpenAI API）** — 目前主要是规则模式（mock），AI模式需要配置OPENAI_API_KEY
6. **15秒脚本** — 现有时长较短时信息密度不够，可优化压缩逻辑
7. **脚本多样性** — 同一输入多次生成的结果变化不够大
8. **SSL** — DigitalOcean代理目前是HTTP，生产推荐加SSL
