# 宏达新媒体脚本工厂 — 脚本生成逻辑完整说明

## 一、整体架构

```
用户操作（向导 4 步）
    ↓
ScriptGeneratorWizard.tsx
    ↓
runPipeline() ← script-pipeline.ts（核心流水线）
    ↓
generateScriptStrategy() → 策略卡
    ↓
buildScriptByDuration() → 15s / 30s / 60s 变体
    ↓
removeAiTone() → 去 AI 味
    ↓
scoreScript() → script-scoring.ts（8 维评分）
    ↓
checkScriptRisk() → 风险检查
    ↓
生成结果展示（3 个变体供选择）
    ↓
确认 → 加入工作台
```

如果配置了 AI_PROVIDER=deepseek 且 DEEPSEEK_API_KEY 有效：
```
用户操作 → callAI() → DeepSeek API → 响应 → 解析 JSON → 展示
```
AI 模式下绕过规则引擎，直接由大模型生成。

## 二、输入阶段（4 步向导）

### 第 1 步：账号平台
选择账号（自动代入人设、定位、目标客户、转化目标）和发布平台（视频号/抖音）

### 第 2 步：内容定位
输入：
- **产品/工艺**（如：PE材质热转印、花膜结构、防背粘工艺）
- **客户痛点**（如：客户问多少钱、PE能不能做热转印、打样和大货不一致）
- **材质**（可选：PE/PP/ABS/PET/不锈钢等）

系统自动检测：
- 如果主题太宽泛（如"客户常见问题一次性说清楚"）→ 自动拆成 5-10 个子选题供选择
- 实时预览开头钩子

### 第 3 步：脚本参数
- **视频时长**：15秒（80-120字，1个核心点）/ 30秒（150-220字，2个）/ 60秒（280-420字，3个）
- **脚本结构**：系统自动选择
- **出镜方式**、**语气风格**

### 第 4 步：参考知识
从知识卡中选择相关知识，知识卡内容会：
1. 影响 getChunkType 判断（决定用哪套模板）
2. core_conclusion 会被检索并参与生成
3. risky_expressions / safer_alternatives 会用于风险检查

## 三、策略卡生成（generateScriptStrategy）

生成脚本前先输出策略卡，包含：

| 字段 | 说明 |
|------|------|
| topic | 脚本主题 |
| hook | 开头钩子（≤25字） |
| hookType | 钩子类型（冲突/提问/反常识/场景/数据） |
| targetCustomer | 目标客户 |
| customerPain | 客户痛点 |
| corePoint | 核心观点（只讲一个） |
| whyWatch | 用户为什么愿意看 |
| solveWhat | 解决什么问题 |
| conversionGoal | 转化目标 |
| risksToAvoid | 需规避的风险 |

策略卡生成逻辑：
1. 调用 splitBroadTopic() 判断是否宽泛主题，是则拆解
2. 调用 generateHook() 生成钩子
3. 从账号信息提取目标客户、转化目标
4. 从知识卡提取核心结论

## 四、钩子生成（generateHook）

### 规则
- 直接点出客户问题
- ≤ 25 个中文字
- 不用泛泛铺垫
- 最好包含数字、材料、场景、冲突或反常识

### 12 类钩子模板（50+ 个钩子）

| 类别 | 示例 |
|------|------|
| pre_quote | "500个杯子，真的不能做热转印吗？" |
| material | "PE瓶能不能做热转印？这句话不能直接回答。" |
| test | "你问我会不会掉，我最怕直接回答'不会'。" |
| color | "客户说颜色按图片做，这个风险很大。" |
| sample | "打样和大货做到一模一样，真的那么简单吗？" |
| small_batch | "500个小批量，适不适合热转印？" |
| customer_pitfall | "客户只发图片，为什么判断不了能不能做热转印？" |
| process | "标准热转印花膜有几层？不是只看图案那么简单。" |
| anti_back | "防背粘只是多加一张纸？没那么简单。" |
| boss_experience | "做了20年印刷，我告诉你最大的坑是什么。" |
| factory_shot | "今天带你看一下热转印花膜是怎么做出来的。" |
| comment_qna | "评论区有人问：PE瓶到底能不能做热转印？" |

### 输入感知
如果有用户输入（pain/product），钩子会自动带入用户的具体内容，不重复同样的话。

## 五、模板类型匹配（getChunkType）

根据输入自动判断使用哪套模板，判断优先级：

1. **templateType** 参数（如果用户显式选择）
2. **source_type**（comment_qna / factory_shot / teardown 等）
3. **知识卡内容**（core_conclusion 和 title 中的关键词）
4. **客户痛点关键词**（包含"价格"→pre_quote，"材质"→material 等）
5. **账号人设**（老板→boss_experience，顾问→pre_quote，技术→process）
6. 默认 → pre_quote

## 六、脚本生成（buildScriptByDuration）

### 字数硬限制
- 15 秒：80-120 汉字，1 个核心点
- 30 秒：150-220 汉字，最多 2 个核心点
- 60 秒：280-420 汉字，最多 3 个核心点

### 生成逻辑
每类模板有 5 个构建块（每个构建块有多个变体）：
```
hook   — 钩子（3-5 个变体）
reason — 核心原因（2-3 个变体）
scene  — 客户场景（2 个变体）
action — 下一步动作（2 个变体）
cta    — 转化引导（1-2 个变体）
```

不同时长的组装方式：
```
15 秒: hook + reason + action
30 秒: hook + reason + scene + action
60 秒: hook + reason + scene + action + cta
```

### 输入感知
- 会根据用户输入的 pain/product 自动选择不同的变体组合（基于输入内容哈希值）
- 如果用户输入了具体痛点，脚本开头会增加一行个性化开场白
- 模板中的 {pain} / {product} / {material} 占位符会被替换为用户输入

### 12 套行业化模板

| 模板 | 适用场景 |
|------|----------|
| pre_quote | 客户问价、报价前收资 |
| material | 材质能不能做热转印 |
| test | 附着力测试、会不会掉 |
| color | 颜色匹配、色差 |
| sample | 打样判断、样品确认 |
| small_batch | 小批量决策 |
| customer_pitfall | 客户误区、常见坑 |
| process | 工艺科普 |
| anti_back | 防背粘问题 |
| boss_experience | 老板经验分享 |
| factory_shot | 工厂实拍解说 |
| comment_qna | 评论区答疑 |

## 七、去 AI 味（removeAiTone）

删除 30+ 种禁止表达：

**通用废话**：「很多客户问我这个问题」「今天统一回答一下」「今天给大家讲一下」「最近很多朋友问」「大家都知道」「在热转印行业中」「随着市场发展」

**文章式连接词**：「首先」「其次」「最后」「综上所述」「显而易见」

**营销空话**：「有效提升」「赋能」「助力」「一站式」「全方位」「闭环」「矩阵」

**过度营销**：「关注我」「点赞收藏」「免费领取」「点击下方链接」「我相信大家」

## 八、评分引擎（scoreScript）

### 8 维评分（满分 100）

| 维度 | 满分 | 评分依据 |
|------|------|----------|
| 开头钩子 | 20 | 是否有具体客户问题或冲突？ |
| 单点聚焦 | 15 | 是否只讲 1 个核心点？ |
| 口语化程度 | 20 | 句子短？有口语连接词？ |
| 客户痛点清晰度 | 15 | 是否明确点出客户问题？ |
| 工厂判断逻辑 | 10 | 有没有"因为/所以/判断"等推理？ |
| 时长控制 | 10 | 字数是否在限制内？ |
| 转化动作 | 5 | 结尾是否有下一步引导？ |
| 风险表达安全 | 5 | 有没有乱承诺？ |

### 硬扣分项
- 通用废话开头 → -20
- 第一/第二/第三枚举 → -5/个
- 像文章不是口播 → -20
- 没有 CTA → -10
- 超字数 → -15
- 高风险承诺 → -15

### 推荐状态
| 分数+风险 | 推荐状态 |
|-----------|----------|
| ≥85 且低风险 | 待审核 |
| 70-84 | 草稿 |
| 60-69 | 需重写 |
| <60 | 不建议保存 |

## 九、风险检查（checkScriptRisk）

检测 7 种风险模式：

| 风险 | 示例 |
|------|------|
| 乱承诺能做 | 一定能做、肯定能做、保证能做 |
| 乱承诺附着力 | 保证不掉、绝对不会掉 |
| 报具体价格 | \d+元、\d+块钱 |
| 承诺交期 | \d+天交货、一定交期 |
| 乱承诺颜色 | 颜色一模一样、百分之百还原 |
| 绝对化表达 | 最能、最好、最专业、最便宜 |
| 附着力乱承诺 | 永不掉、永远不掉 |

## 十、知识检索（retrieveKnowledgeForScript）

生成脚本前会检索知识卡：
1. 从已确认的知识卡中检索
2. 返回 core_conclusion / risky_expressions / safer_alternatives
3. 影响模板选择（getChunkType）
4. 用于风险检查

## 十一、AI 模式（DeepSeek / OpenAI）

如果配置了 AI_PROVIDER=deepseek（或 openai），规则引擎被绕过，脚本完全由 AI 生成：

### 配置方式
```
AI_PROVIDER=deepseek
DEEPSEEK_API_KEY=sk-xxx
```

### 生成流程
```
用户输入 → callAI() → 调用 getProvider()
  → DeepSeekProvider.generateStructured()
    → DeepSeek API (chat completions)
      → 返回 JSON（策略卡 + 脚本 + 评分）
```

### 输出格式
AI 同样输出结构化 JSON（跟规则引擎一样的格式）：
```
{
  strategy: { topic, hook, targetCustomer, ... },
  variants: [{ duration, hook, script, wordCount, score, ... }],
  risk: { riskLevel, riskPoints },
  recommendedStatus
}
```

## 十二、三栏工作台

生成后的脚本进入三栏工作台：

| 栏 | 功能 |
|------|------|
| 左栏 | 脚本列表 + 搜索 + 多选 |
| 中栏 | 脚本编辑器（可微调） |
| 右栏 | AI操作、评分卡、风险、来源追溯、推进选题库 |

### 后续操作
- **推进选题库** → 推送到选题库待审核
- **编辑** → 微调脚本文字
- **AI 操作** → 12 种润色（更口语/更专业/更短/查风险等）
- **批量操作** → 批量润色/评分/风险检查/删除
